# Day 5 - Super Key, Unique Key, Natural Key and Surrogate Key

## What is a Super Key?

A Super Key is any column or combination of columns that can uniquely identify a row.

Examples:
- StudentID
- Email
- StudentID + Name
- Email + Name

A Super Key may contain extra unnecessary columns.

## What is a Candidate Key?

A Candidate Key is a minimal unique identifier.

Examples:
- StudentID
- Email

Every Candidate Key is a Super Key.

## What is a Unique Key?

A Unique Key prevents duplicate values in a column.

Characteristics:
- No duplicate values allowed
- Usually allows one NULL value
- Multiple Unique Keys can exist in a table

## What is a Natural Key?

A Natural Key is a real-world identifier that already exists outside the database.

Examples:
- Aadhaar Number
- Email
- Passport Number

These values have business meaning.

## What is a Surrogate Key?

A Surrogate Key is an artificial identifier generated by the database.

Examples:
- StudentID
- UserID
- EmployeeID

These values have no business meaning.

## Natural Key vs Surrogate Key

Natural Key:
- Exists naturally
- Has business meaning

Examples:
- Email
- Aadhaar Number

Surrogate Key:
- Artificially created
- No business meaning

Examples:
- StudentID
- UserID
