# Day 2 - Tables, Rows, Columns, Records and Schema

## What is a Table?

A table is a collection of related data organized into rows and columns.

Example:

| StudentID | StudentName | Department |
| --------- | ----------- | ---------- |
| 101       | Varsha      | AI&DS      |
| 102       | Rahul       | CSE        |

The entire structure is called a table.

## What is a Column?

A column represents a specific attribute of data.

Examples:

* StudentID
* StudentName
* Department

Columns are arranged vertically.

## What is a Row?

A row contains all information about one record.

Example:

101 | Varsha | AI&DS

This complete line is one row.

## What is a Record?

A record is another name for a row.

Row = Record

## What is a Schema?

A schema is the blueprint or structure of a database.

It defines:

* Tables
* Columns
* Data Types
* Relationships

## Row vs Column

Row:

* Horizontal
* Represents one record

Column:

* Vertical
* Represents one attribute
